# 同步流程

## 1. 节点启动

1. 加载本地配置
2. 初始化数据库
3. 获取本机 PeerID
4. 订阅 PubSub topic
5. 广播节点 announce
6. 启动目录监听
7. 启动 Web API / UI

## 2. 本地文件变更

1. chokidar 检测到 add 或 change
2. 对内容做可选加密
3. 写入 IPFS，得到 CID
4. 更新 files 表和历史版本
5. 向互信节点广播 file-changed

## 3. 远端变更处理

1. 校验发送方是否互信
2. 校验 syncId 是否存在
3. 对比本地版本与远端版本
4. 使用冲突解决规则判断胜者
5. 拉取内容、解密、落盘并更新索引

## 4. 信任建立

1. 用户在界面将节点设为 trusted
2. 本地写库并通知对方
3. 双方互信后触发 state-sync
4. 交换已有文件状态并补齐缺失内容
